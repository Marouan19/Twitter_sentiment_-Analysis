Mini Projet: Prédiction de sentiments en temps réel des flux de données de réseau social Twitter
A présenter : Lundi 13 Mai 2024 par trinôme

Il s’agit de développer une application Web basée sur l’API Apache KAFKA Stream, permettant de faire l’analyse des flux en Twitter «Twitter Real Time Sentiment Analysis» afin de prédire le type de sentiment en temps réel.

Vocabulaire Apache Kafka
  • Topic: Un flux de messages appartenant à une catégorie particulière. Les données sont stockées dans des topics.
  • Partitions: Chaque topic est divisé en partitions. Pour chaque topic, Kafka conserve un minimum d'une partition. Chaque partition contient des messages dans une séquence ordonnée immuable. Une partition est implémentée comme un ensemble de segments de tailles égales.
  • Brokers: Les brokers (ou courtiers) sont de simples systèmes responsables de maintenir les données publiées. Chaque courtier peut avoir zéro ou plusieurs partitions par topic.
  • Producers: Les producteurs sont les éditeurs de messages à un ou plusieurs topics Kafka. Ils envoient des données aux courtiers Kafka. Chaque fois qu'un producteur publie un message à un courtier, ce dernier rattache le message au dernier segment, ajouté ainsi à une partition. Un producteur peut également envoyer un message à une partition particulière.
  • Consumers: Les consommateurs lisent les données à partir des brokers. Ils souscrivent à un ou plusieurs topics, et consomment les messages publiés en extrayant les données à partir des brokers.
Architecture à implémenter
  1. Lancer puis lire en temps réel les données du fichier twitter_training.csv en utilisant Apache KAFKA Streams
  2. Faire des prétraitements nécessaires sur les données du fichier en utilisant la librairie Pyspark Mlib, NLTK, etc.
  3. En utilisant des modèles Machine Learning supervisés (Logistic Regression, XGBoost, etc.), faire un entrainement sur la base d’apprentissage twitter_training.csv (69491 tweets).
  4. Enregistrer le meilleur modèle sous format .pkl
  5. En utilisant le modèle pré-entraîné et sauvegardé, prédire en temps réel si le tweet est classé (Negative, Positive, Neutral ou Irrelevant) sur les données de validation twitter_validation.csv
  6. Une sauvegarde des flux de sortie en Base de données MongoDB est nécessaire pour la journalisation des résultats (Ceci va servir à la présentation des résultats en Front-End).
  7. L’exploitation de l’architecture sera à travers une application Web, permettant l’exploitation des données de test, ainsi que la présentation des résultats de traitement sauvegardés sur la base MongoDB.
  8. Uploader le projet sur le réseau GitHub.
Outils de travail
  • Librairies: Apache Kafka Stream, Spark Mlib, NLTK, Pandas, Matplotlib, etc.
  • Frameworks: Flask, Django
  • Langages: Python, Java, JavaScript, etc.
  • Éditeurs: IntelliJ IDEA
  • Systèmes d’exploitation: Unix, MacOS ou Windows
Description des données

Chaque ligne de la base d’apprentissage twitter_training.csv représente un Tweet. Les types de données des caractéristiques (Features) sont :

  • Tweet ID: int
  • Entity: string
  • Sentiment: string (Label)
  • Tweet content: string

La base de validation «twitter_validation.csv» contient 998 lignes (Tweets) et 4 colonnes (caractéristiques). La colonne « Sentiment » est la cible à prédire (Label).

Source de données: https://www.kaggle.com/datasets/jp797498e/twitter-entity-sentiment-analysis
